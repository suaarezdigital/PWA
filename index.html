03🎯 Por qué decidimos investigar

Banco Azteca está explorando una nueva funcionalidad que permitiría compartir un enlace de pago, para que otra persona pueda realizar directamente el pago de un préstamo — sin pasar primero por una transferencia SPEI.

Esta investigación busca entender si esa propuesta realmente puede modificar el comportamiento que acabamos de describir, o si el hábito actual es demasiado fuerte para cambiarlo.

🧩 Patrón observado

El 86.7% de los participantes (13 de 15) paga personalmente su préstamo, incluso cuando alguien más les da el dinero. El mismo circuito — solicitar, recibir, pagar, confirmar — se repite en la mayoría de las entrevistas. No estamos introduciendo una funcionalidad en un espacio vacío: estamos intentando modificar un comportamiento que ya satisface la necesidad de los usuarios.

🔎 Insight

El comportamiento actual satisface una necesidad de control y certeza; el enlace deberá ofrecer esas mismas garantías para convertirse en una alternativa viable.

🎯 Implicación para Producto

El enlace debe transmitir el mismo nivel de control y certeza que hoy ofrece el proceso manual, o no logrará desplazarlo.

🧩 Patrón observado

Menos de la mitad (7 de 15) señaló espontáneamente una preocupación de confianza, seguridad o privacidad. No hay un solo tema dominante: se reparte entre privacidad de la deuda, verificación del enlace, seguridad de datos y confirmación del pago.

🔎 Insight

La confianza es la preocupación más citada, pero no es unánime ni concentrada en un solo motivo — el enlace debe atender varios frentes a la vez (privacidad, verificación y confirmación), no solo uno.

🎯 Implicación para Producto

Reducir incertidumbre y hacer evidente el estado del pago en cada paso, antes de solicitar cualquier acción del usuario.
